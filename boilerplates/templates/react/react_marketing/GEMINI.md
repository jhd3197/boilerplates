# Project Intent

## Purpose
Provide project intent and behavioral expectations for Antigravity and Google models.

## Scope
Model neutral guidance. No permissions. No Claude specific instructions.

## One Paragraph Project Summary
This is a modern React marketing website boilerplate built with Vite, featuring SEO optimization and reusable components for building landing pages and promotional sites.

## What the Project is For
To serve as a starting template for building React marketing websites, providing essential setup for development, building, and deployment with components for hero sections, features, and footers.

## Stability Guarantees
- Uses stable versions of React and Vite
- Includes SEO with react-helmet-async
- Follows standard marketing site structures

## Data Contracts and Invariants
- Components expect standard React props
- File structure adheres to src/ with App.jsx, main.jsx, index.css, components/
- Uses react-helmet-async for meta tags

## Compatibility Expectations
- Compatible with modern browsers and Node.js 16+
- Follows web standards
- SEO-friendly

## Risk Tolerance Level
Low to medium - prefer proven libraries and avoid experimental features.

## What Changes are Considered Safe
- Adding new components or sections
- Updating documentation
- Minor style adjustments
- Bug fixes in existing code
- Updating content and meta tags

## What Changes Require Explicit Approval
- Major architectural changes
- Upgrading major dependencies
- Restructuring folders
- Changing SEO strategy

## Non Goals
- Not intended for complex applications
- Not for server-side rendering
- Avoids routing by default

## Important Notes
Antigravity will also analyze README.md and other markdown documentation. This file clarifies intent, sets boundaries, and avoids duplicating documentation.

## Outcome
Antigravity builds a correct mental model of the project before acting.
